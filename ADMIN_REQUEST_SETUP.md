# Admin Request Feature - Setup Guide

## Overview
This feature allows users to request admin access to the pharmacy system. The requests are sent via email to the admin for approval or rejection.

## Setup Instructions

### 1. Gmail Configuration (Required for email sending)

To send emails through Gmail, you need to:

#### Step 1: Enable 2-Factor Authentication
1. Go to Google Account: https://myaccount.google.com/
2. Click "Security" in the left sidebar
3. Enable 2-Step Verification if not already enabled

#### Step 2: Create an App-Specific Password
1. Go back to Security settings
2. Look for "App passwords" (appears only if 2FA is enabled)
3. Select "Mail" and "Windows Computer"
4. Google will generate a 16-character password
5. Copy this password

#### Step 3: Update .env file
In `backend/.env`, add/update these values:

```dotenv
# Email Configuration (Gmail)
SMTP_SERVICE=gmail
SMTP_EMAIL=preethisampath123d@gmail.com
SMTP_PASSWORD=your_16_character_app_password_here
ADMIN_EMAIL=preethisampath123d@gmail.com
APP_URL=http://localhost:3000
```

**Important:** Replace `your_16_character_app_password_here` with the actual password generated by Google.

### 4. Workflow

#### User Side:
1. Go to Register page
2. Click "Request Admin Access" tab
3. Fill in form with reason for requesting admin access
4. Submit request
5. User receives confirmation email
6. When approved, they receive approval email with login instructions

#### Admin Side:
1. Navigate to "Admin Requests" page (admin only)
2. View pending requests
3. Click "Approve" or "Reject"
4. If rejecting, optionally provide rejection reason
5. User receives approval/rejection email

### 5. Email Templates

#### Approval Request Email (sent to ADMIN_EMAIL)
- Includes user information
- Shows the reason provided
- Contains approve and reject links

#### Approval Confirmation Email (sent to user)
- Notifies user that their request was approved
- Provides login information
- Lists admin privileges

#### Rejection Email (sent to user)
- Notifies user that request was not approved
- Includes rejection reason if provided
- Suggests contacting admin for more information

### 6. API Endpoints

#### Public Endpoints
- `POST /api/v1/auth/request-admin` - Submit admin request
- `PUT /api/v1/auth/admin-requests/:token/approve` - Approve request (public link from email)
- `PUT /api/v1/auth/admin-requests/:token/reject` - Reject request (public link from email)

#### Protected Endpoints (Admin Only)
- `GET /api/v1/auth/admin-requests?status=pending` - List admin requests (filtered by status)

### 7. Database Model

AdminRequest schema includes:
- email (unique)
- firstName, lastName
- password (hashed)
- reason (for admin access)
- status (pending, approved, rejected)
- approvalToken (unique token for approval/rejection links)
- rejectionReason (optional)
- requestedAt, respondedAt
- respondedBy (admin who responded)

### 8. Troubleshooting

#### Emails not sending:
1. Check if SMTP credentials in .env are correct
2. Verify Gmail SMTP_PASSWORD is the app-specific password, not regular password
3. Enable "Less secure app access" if still not working (alternative method)
4. Check backend logs for email service errors

#### Admin approval token invalid:
- Tokens expire after the request is processed
- Each request gets a unique token
- Tokens cannot be reused

#### User not created after admin approval:
- Check database connection
- Verify all required fields are populated
- Check backend logs for errors

### 9. Security Notes

- Approval tokens are unique and single-use
- Passwords are hashed before storage
- Admin emails are not exposed to public
- SMTP password should never be hardcoded (use .env)
- Approval links work without authentication (to allow offline approval by admins)

## Example Usage

### Requesting Admin Access (Frontend)
```javascript
import authService from '../services/authService';

await authService.requestAdminAccess({
  email: 'user@example.com',
  password: 'securePassword123',
  name: 'John Doe',
  reason: 'I am the pharmacy manager and need admin access to manage users and reports'
});
```

### Backend Response
```javascript
{
  status: 'success',
  message: 'Admin access request submitted. Please check your email for confirmation.',
  data: {
    requestId: '507f1f77bcf86cd799439011'
  }
}
```
